# BashPass - Secure Password Manager

A lightweight, encrypted password manager written in pure Bash with AES-256-CBC encryption.

## Features

- **Strong Encryption**: AES-256-CBC with PBKDF2 key derivation (100,000 iterations)
- **Master Password**: Single master password protects all your credentials
- **Secure Password Generation**: Generate strong, random passwords
- **Search & Manage**: Find, view, and delete stored passwords
- **Zero Dependencies**: Uses only standard Linux tools (OpenSSL, Bash)

## Installation

```bash
git clone <your-bashpass-repo-url>.git
cd BashPass
chmod +x BashPass
```

## Usage

```bash
./BashPass
```

### First Run
1. Run `./BashPass`
2. Create a master password (enter twice to confirm)
3. Your encrypted vault is created at `password_store.csv.enc`

### Normal Usage
1. Run `./BashPass`
2. Enter your master password
3. Choose an option:
   - Generate a new password
   - View all saved passwords
   - Search for a password
   - Delete a password

## Security Details

| Feature | Implementation |
|---------|----------------|
| Encryption | AES-256-CBC |
| Key Derivation | PBKDF2 with 100,000 iterations |
| Salt | Auto-generated by OpenSSL |
| Master Password | Never stored, verified via decryption |

### How It Works
1. Master password is never stored on disk
2. Passwords are encrypted/decrypted in memory during session
3. Plaintext file is deleted immediately after encryption
4. Vault file (`password_store.csv.enc`) is safe to backup

## Requirements

- **Bash 4.0+** (pre-installed on virtually all Linux systems)
- **OpenSSL** (standard on 99% of Linux distributions)
- Linux/Unix environment (works on Fedora, Ubuntu, Debian, Arch, RHEL, Alpine, etc.)

### Installing Dependencies

All required tools are typically pre-installed. If missing:

```bash
# Fedora/RHEL/CentOS
sudo dnf install openssl

# Ubuntu/Debian
sudo apt install openssl

# Arch/Manjaro
sudo pacman -S openssl

# Alpine
sudo apk add openssl bash

# openSUSE
sudo zypper install openssl
```

## Troubleshooting

### Always asks for new master password

**Check if vault file was created:**
```bash
ls -la password_store.csv.enc
```

**Verify OpenSSL works:**
```bash
echo "test" | openssl enc -aes-256-cbc -salt -pbkdf2 -iter 100000 -pass pass:test -out /tmp/test.enc && echo "OpenSSL OK"
openssl enc -aes-256-cbc -d -pbkdf2 -iter 100000 -pass pass:test -in /tmp/test.enc
```

**Check permissions:**
```bash
chmod 700 BashPass
ls -la password_store.csv.enc
```

### Wrong password errors despite correct password
```bash
# Backup and test decryption manually
cp password_store.csv.enc backup.enc
openssl enc -aes-256-cbc -d -pbkdf2 -iter 100000 -pass pass:YOUR_PASSWORD -in backup.enc
```

### OpenSSL version issues
If using older OpenSSL (<1.1.1), PBKDF2 may not be available:
```bash
openssl version
# If < 1.1.1, remove -pbkdf2 flag from script (less secure)
```

### Encryption fails silently
Edit the script and change `2>/dev/null` to see errors:
```bash
openssl enc -aes-256-cbc -salt -pbkdf2 -iter 100000 -pass pass:"$MASTER_PASSWORD" -in "$PASSWORD_FILE" -out "$ENCRYPTED_FILE"
```

## Backup & Recovery

**To backup:**
```bash
cp password_store.csv.enc /backup/
```

**To restore:**
```bash
cp /backup/password_store.csv.enc .
./BashPass
# Enter your master password
```

## Security Warnings

- Master password cannot be recovered if lost
- Plaintext passwords exist in memory during session
- Use on trusted systems only
- Consider using `pass` or `bitwarden` for production use
